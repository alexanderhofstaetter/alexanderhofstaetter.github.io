        Query options for a Z-PI demographics query.

        Required parameters are:

        • family

        or

        • given
        • date_of_birth

        The parameter gender and address-related parameters are always optional.

        The date_of_birth can be either a value, a partial value, a range of values or a range of partial values.

        Wildcard Parameters

        Wildcard search enables searching by prefix. The following parameters support wildcard search, where W<N> means that at least N characters have to be provided before the wildcard:

        • given W4
        • family W4
        • postal_code W2
        • city W4
        • street W4

        Special Search Opts

        There are the following switches to tune searching by name:

        • phonetic - turns on phonetic matching of the given and family name.
        • additional_names - causes the search by given and family to be matched against all known names of the patient. Default is to only search by the current name.
        • only_patients_alive - only patients not marked as deceased will be returned.
        • all_patients - all identities in a linked group are searched.

        Performs a patient demographic query against the Z-PI.

        Either performs a query by LivingSubject.id when given an identifier query tuple or a query by demographic data. For demographic query options see demographic_query_opt/0.
