        consult_opt()

        @type consult_opt() ::
  {:ssn, String.t()}
  | {:provider, String.t()}
  | {:claim, String.t()}
  | {:afterwards, %{date: Date.t(), reason: String.t()}}
  | {:offline, Date.t()}
  | {:surrogate, String.t()}

        Creating a consult may take the following options:

        • :ssn - specifies the SSN to perform the consult for
        • :provider - specifies a provider in case of a previous claims fault
        • :claim - specifies a claims ID in case of a previous claims fault
        • :afterwards - a map specifying the :date of consult and the :reason for retroactive registering (this would be a tuple, but then it would not be json serializable)
        • :offline - specifies that this consult was recorded offline and is now being retried. This takes the date of the consult.
        • :surrogate - specifies a surrogate voucher code

        When specifying both :offline and :afterwards the dates must be the same. If they are not, the last specified date wins.
