        OWAT

        This is the Elixir implementation for the OWAT Core.

        API

        The API consists of the following parts:

        • Authentication Endpoints
        • Public REST Resources
        • Restricted Websocket Channels

        Authentication

        Authentication is necessary to access restricted functionality.

        The general authentication is done via an OIDC provider. Further information can be obtained at the docs for OwatWeb.AuthController.

        Developers Information

        This repository comes with a docker-compose file to set up a development environment. Run docker compose watch to start a local backend and necessary services. The backend will be available at localhost:4000 and will be automatically reloaded when changes are made to the source code. You will have to make the AZURE_CLIENT_SECRET and EID_CLIENT_SECRET available in the environment for authentication to work.

        To start your Phoenix server on the host machine:

        • Run mix setup to install and setup dependencies and set up git hooks
        • Start Phoenix endpoint with mix phx.server or inside IEx with iex -S mix phx.server

        Now you can visit localhost:4000 from your browser.

        The dev environment must have the following setup:

        • An env-var DB_HOSTNAME pointing to your DB.
        • The DB must have the name owat_ex and the user owat.
        • For SS12 operation a valid microsoap service must be deployed and its script referenced in the MICROSOAP env-var.
        • For search operation a meilisearch instance must be referenced via MEILISEARCH_ENDPOINT and MEILISEARCH_API_KEY.

        For convenience there exists the ShellHelper module with a few supportive functions.

        Ready to run in production? Please check our deployment guides.

        Git Hooks

        We use the git_hooks library to run pre-commit and pre-push checks. If you want to create a commit without running the checks, you can use git commit --no-verify, the same applies to git push. If you get an error message that the hooks are not executable, you can run mix git_hooks.install. Currently, the following checks are run:

        • pre-commit:
          • mix format --check-formatted
        • pre-push:
          • mix dialyzer --plt
          • mix credo --all --ignore design
          • mix test
